# Proposal: optimize-chat-ux

## Why

当前聊天界面和广场页面存在多个体验问题：

1. **加载指示器不美观** — 等待回复期间显示旋转的 `<n-spin>` 图标，且文字输出过程中仍然显示，不符合聊天应用的交互惯例
2. **发送快捷键不灵活** — 固定 Enter 换行 / Ctrl+Enter 发送且不可切换，部分用户偏好 Enter 发送的即时通讯习惯
3. **聊天顶部信息不实用** — 显示模型名称而非角色简介，用户更关心角色设定
4. **广场卡片 tooltip 太宽** — 鼠标悬停显示完整简介时弹框过宽导致显示不全，且默认向上弹出遮挡头像
5. **流式语音** — 尝试过句子级实时 TTS，因 Edge TTS 网络延迟无法做到文字语音同步，并行请求导致播放顺序错乱，已回退

## What Changes

- **打字指示器** — 将 `<n-spin>` 替换为纯 CSS 弹跳三点动画，仅在等待首 token 时显示
- **快捷键互换** — 新增 `sendOnEnter` 开关，Enter 发送为默认，发送按钮旁增加切换入口，持久化
- **聊天顶部** — 模型名称替换为角色简介，单行截断 + hover tooltip
- **广场卡片** — `n-ellipsis` tooltip 限制宽度 280px，弹框方向改为下方
- **流式语音（已回退）** — 恢复为文字完成后 TTS 合成全文

## Scope

- In scope: `Chat.vue`、`CharacterCard.vue` 模板+脚本+样式改动
- Out of scope: 后端改动、WebSocket 升级、TTS 实时流式
